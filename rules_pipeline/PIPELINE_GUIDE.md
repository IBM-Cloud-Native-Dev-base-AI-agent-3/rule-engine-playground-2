# 🛠️ PIPELINE_GUIDE: 룰 엔진 데이터 파이프라인 가이드

이 파일은 룰 엔진 파이프라인 폴더(`rules_pipeline`) 내부의 핵심 DB 스크립트와 가동 파일들의 역할 및 상호 유기적 구조를 자세히 설명합니다.

---

## 🗂️ 파일별 세부 역할 소개

이 폴더 안의 파일들은 공고 데이터 파싱부터 설계도 그리기, 규칙 연동, 실시간 자동 검증 및 가독성 높은 엑셀 명세서 추출에 이르기까지 **행정 규정의 데이터화 및 룰 엔진 코어**의 중추 역할을 수행합니다.

### 1. `db_helper.py` (MySQL 연동 및 세딩 엔진)
* **주요 역할:** 파이썬 환경과 로컬 MySQL 데이터베이스(`rule_engine`) 간의 접속 통로이자 복원 엔진입니다.
* **상세 기능:**
  * 프로젝트 루트의 환경 변수 파일(`.env`)을 로드하여 안전하게 MySQL 계정 정보를 추출합니다.
  * 새로운 컴퓨터나 빈 서버 환경에서 실행 시, 테이블 설계도(`schema.sql`)를 돌려 **자동으로 테이블 및 검색 최적화 인덱스를 구축**합니다.
  * 백업 파일(`insert_rules.sql`)을 파싱하여 **188개의 희망하우징 룰을 원클릭으로 주입**해 줍니다.

### 2. `schema.sql` (데이터베이스 설계 도면 - DDL)
* **주요 역할:** 룰 엔진 시스템의 5대 테이블 뼈대와 제약조건을 규정합니다.
* **상세 기능:**
  * 공고 제공자(`provider`), 대분류 카테고리(`category`), 모집 공고(`announcement`), 상세 모집 전형(`recruitment_target`), 개별 통제 규칙(`eligibility_rule`)의 5개 테이블을 관계형(RDBMS)으로 매핑합니다.
  * 부모 공고가 지워지면 아래 소속된 모든 세부 규칙들도 함께 자동 파괴되도록 **외래키 종속 삭제(`ON DELETE CASCADE`)** 제약조건이 정교하게 걸려 있습니다.
  * 규칙을 빠른 속도로 쿼리할 수 있도록 인덱스 최적화가 이식되어 있습니다.

### 3. `test_rules.py` (라이브 MySQL 동적 룰 검증기)
* **주요 역할:** 룰 엔진이 실물 행정 규정에 맞춰 오차 없이 완벽하게 판정해 내는지 가상 검증합니다.
* **상세 기능:**
  * 하드코딩 조건식 대신, 특정 타겟 ID(예: `연남 남성 3순위`)를 주면 **MySQL에서 실시간으로 규칙 정의를 로드**합니다.
  * 3대 모의 시나리오(자격 합격군, 성별 미치 불일치 탈락군, 일반 대학원생 탈락군, 자가용 보유 위반 탈락군, 자산 초과 탈락군 등)를 대입해 동적으로 평가한 뒤 합격여부와 사유를 예쁘게 출력합니다.

### 4. `export_rules_to_excel.py` (라이브 룰 엑셀 리포트 빌더)
* **주요 역할:** MySQL 내부의 복잡한 룰 관계 데이터를 사람이 읽을 수 있는 가장 깔끔한 엑셀 보고서로 렌더링합니다.
* **상세 기능:**
  * MySQL에 접속해 Join 쿼리를 돌려 실시간 전체 규칙 리스트를 긁어옵니다.
  * 가점 항목의 점수(`|2`, `|3` 등)를 문자열에서 자동 분리하여 '배점' 열로 정제합니다.
  * 한글 폰트 폭을 감안한 **열 너비 자동 맞춤(Autofit)**을 지원하여 즉시 보고용으로 쓸 수 있는 미려한 가독성의 [`all_rules.xlsx`](file:///c:/Users/hi/Desktop/rule-engine-playground-2/data/processed/all_rules.xlsx) 파일을 최종 생성합니다.

### 5. `extract_rules.py` (규칙 추출 빌드 유틸리티)
* **주요 역할:** 원시 공고 PDF의 텍스트 요약본으로부터 룰 모델을 설계하고 대량의 SQL 삽입 구문을 도출해 낸 메인 컴파일러 코드입니다.

### 6. `query_db_debug.py` (구조 테스트용 디버깅 스크립트)
* **주요 역할:** 데이터베이스의 테이블 연결 상태나 규칙 레코드들의 탑재 개수를 터미널 상에서 가볍게 카운트하여 모니터링하기 위한 경량 디버깅 보조 유틸리티입니다.

## 🔄 데이터 흐름도 (Data Pipeline Flow)

규칙들이 새롭게 수정되거나 배포될 때의 유기적인 연동 흐름은 다음과 같습니다.

```text
  [ raw_pdf 가이드라인 ]
           │
           ▼ (extract_rules.py 컴파일러 가동)
  [ insert_rules.sql 백업 데이터 생성 ]
           │
           ▼ (db_helper.py 원클릭 세딩 실행)
  [ 🗄️ 로컬 MySQL rule_engine 라이브 서버 적재 ]
     │                                │
     ▼ (test_rules.py 실시간 검증)     ▼ (export_rules_to_excel.py 구동)
  [ 🧠 동적 규칙 엔진 판정 ]       [ 📊 미려한 Excel 명세서 all_rules.xlsx 자동 갱신 ]
```
